🏆 Inference Collapse: Real-Time Hallucination Audit
Inference Collapse is a cyberpunk-inspired real-time simulation game that binds Gemma 4 (31B-it) inference outputs directly to downstream world-state physics and behavior.

The project visualizes LLM hallucination, overconfidence, and unstable reasoning not merely as textual errors, but as physical threats, spatial corruption, and environmental instability inside an interactive game system.

💡 Project Overview
In this simulation, the player acts as an Audit Bot (🤖) exploring corrupted infrastructure sectors while monitoring inference signals generated by Gemma 4.

As reasoning quality degrades, the game world itself becomes unstable. Structured inference signals from Gemma 4 are directly injected into gameplay logic in real time:

confidence Sync: Unverified or hallucinated confidence scales the enemy AI (👤) velocity and Field of View (FOV) while drastically narrowing the player's visibility radius.

severity Sync: High fallacy severity scores (1–5) directly amplify spatial coordinate translation noise, triggering intense screen glitches and coordinate jitter.

Rather than treating hallucination as a passive model failure, this project turns it into an interactive gameplay mechanic where LLM reasoning quality dictates the dynamic difficulty system of the game.

🧠 Why Gemma 4 (31B-it)?
This project intentionally uses Gemma 4 – 31B-it (Dense) due to three key architectural advantages:

High-Fidelity Reasoned Hallucinations: To simulate believable flawed logic, the model needs to maintain coherent reasoning frameworks while tracking biased or false clues. The 31B-it model strikes an incredible balance, producing highly plausible "detective-like" contradictions rather than simple random errors.

Reliable Structured JSON Outputs: The game mechanics tightly bind inference values directly into real-time JavaScript physics loops. Gemma 4’s robust adherence to response_mime_type="application/json" guarantees stable data parsing under adversarial conditions without markdown code-block wrapping failures.

Local Execution Blueprint: While the current entry utilizes API-driven inference, the resource profile and performance of the 31B Dense architecture allow for a direct future migration to fully local, edge-based deployment on developer workstations.

🛠 Technical Implementation
Architecture Matrix
Backend: Python | Streamlit | Google GenAI SDK (google-genai)

Parallel execution triggers both sharp deductive reasoning and forced logical fallacies using specific systemic condition prompts.

Frontend: HTML5 Canvas | JavaScript

Gemma's structured data is injected into the web context via streamlit.components.v1, mapping LLM states into a 60fps rendering and vector calculation loop (requestAnimationFrame).

Inference Pipeline
[Raw Evidence Log] ➔ [Gemma 4 Inference Loop] ➔ [Structured JSON Output] ➔ [Downstream World-State Binding] ➔ [Real-Time Canvas Physics Update]
Production-Grade Fallback Mode
If an API key is missing or rate limits are reached, the engine automatically enters a robust Local Simulation Fallback Mode. It serves synthesized backup inference vectors so judges can experience the exact relationship between the LLM logic and world-state degradation under any environment.

🎨 Originality
Hallucination as a Gameplay Mechanic: Translates the abstract concept of LLM reliability into mechanical gaming threats (Enemy Buffs, Fog of War, and Coordinate Distortion).

Minimalist Retro Aesthetics: Features a high-contrast terminal theme inspired by 80s sci-fi interfaces and minimalist cognitive layouts, minimizing UI bloat while maximizing structural clarity.

🚀 Setup
1. Requirements & Installation
Clone the repository and install the required dependencies:


git clone <your-repository-url>
cd inference-collapse
pip install streamlit google-genai python-dotenv
2. API Configuration
Create your own API key from Google AI Studio. You can store your key using either a .env file or Streamlit secrets:

Option A: .env file (Root Directory)


API_KEY=YOUR_GEMINI_AI_STUDIO_API_KEY
Option B: Streamlit Secrets (.streamlit/secrets.toml)


API_KEY = "YOUR_GEMINI_AI_STUDIO_API_KEY"
(Note: If no API key is detected, the system will seamlessly run in Local Inference Fallback Mode for demonstration purposes.)

3. Run the Application

streamlit run app.py
