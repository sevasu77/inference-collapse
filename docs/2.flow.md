🧠 2. 各フェーズの意味
① UI入力（Player Action Layer）

役割：

プレイヤーが世界へ干渉する唯一の入口

入力例：

証拠テキスト
調査対象選択
行動コマンド
② LLM推論（Cognition Layer）

役割：

世界を「解釈するAI」
真偽・危険度・矛盾を評価

出力構造：

{
  "report": "推理内容",
  "confidence": 0.0 ~ 1.0,
  "severity": 0.0 ~ 1.0,
  "contradiction": true / false
}
③ State更新（World Memory Layer）

役割：

世界の「記憶」を更新する
LLMの結果を安全に格納

重要設計：

直接エンジンを書き換えない
必ず State を経由する

State分類：

WorldState（現実）
AIState（推論）
EngineState（物理）
MetaState（ログ）
④ シミュレーション（Physics Layer）

役割：

世界の物理法則を再計算

入力：

State
Threat Level

出力：

次フレームの世界状態

変換ルール：

confidence ↑ → 敵速度↑
severity ↑ → グリッチ強化
hallucination ↑ → 視界（FOV）縮小
⑤ 描画（Presentation Layer）

役割：

プレイヤーに世界を見せる

構成：

Canvas（ゲーム画面）
Streamlit UI（管理画面）
インジケータ（状態表示）
🔁 3. 全体ループの本質

このシステムの本質はこれ：

「AIの認知 → 世界の物理法則へ変換 → プレイヤー体験」

つまり：

AIの思考 = ゲーム難易度

従来ゲーム：

difficulty = static parameter

本システム：

difficulty = AI state (confidence / hallucination / severity)
⚠️ 4. 設計上の重要制約
❌ 禁止構造（危険）
LLM → Game Engine（直結）
UI → State（直書き）
JS → AI state（直接操作）

理由：

再現性崩壊
デバッグ不能
世界の破綻
✅ 正しい構造
UI → API → LLM → State → Engine → Render

必ず一方向フローにする

🧩 5. このフローの意味（設計思想）

このシステムは単なるゲームではなく：

「AIの認知状態を物理現象として観測する実験装置」

である。

つまり：

LLM = 思考
State = 記憶
Engine = 物理法則
UI = 観測装置
🧠 最後に（重要）

このフローは固定ではなく、将来的には：

async inference
event-driven state
distributed engine

へ進化可能な「最小コア構造」として設計されている。
