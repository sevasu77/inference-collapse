Inference Collapse: Real-Time Hallucination Audit System
1. Overview（概要）

本プロジェクトは、LLM（Gemma 4）の推論結果を単なるテキスト出力として扱うのではなく、
“認知結果が物理世界に影響を与えるシミュレーション空間”として再構成した実験的ゲームシステムである。

従来の構造：

LLM → 応答生成 → UI表示

本システム：

LLM → 認知解釈 → 物理パラメータ変換 → 世界状態変化 → UI

つまり本質はゲームではなく、

Inference（推論）と World Dynamics（世界生成）の結合実験

である。

2. Core Design Philosophy（設計思想）

本プロジェクトは以下の原則に基づいて構築されている：

2.1 Prototype First Principle（プロトタイプ優先原則）

初期段階では「正しい設計」よりも以下を優先する：

動作すること
現象として成立すること
LLMが世界に影響する体験の検証
2.2 Post-Hoc Architecture Formation（後付け構造化）

本システムは最初から設計されたものではなく：

動作する混沌から構造を抽出する方式

で構築されている。

2.3 Inference-to-Physics Mapping（推論→物理変換）

LLM出力は単なるテキストではなく：

confidence → 視界・追跡精度
severity → 空間ノイズ・歪み
contradiction → 不確実性エントロピー

として解釈される。

3. System Architecture（全体構造）

本システムは以下のパイプラインで構成される：

LLM (Gemma 4)
   ↓
Decision Layer（軽量正規化・意味付け）
   ↓
Cognitive State（認知結果構造）
   ↓
Threat Engine（物理変換層）
   ↓
Simulation Engine（ゲーム世界）
   ↓
Rendering Layer（Canvas UI）
4. Core Components（構成要素）
4.1 LLM Layer（Gemma 4）

役割：

セキュリティログの解析
矛盾検出
推論生成

出力形式：

{
  "report": "...",
  "confidence": "90%",
  "suspicion": "HIGH",
  "contradiction": "...",
  "severity": 1-5
}

重要な設計意図：

LLMは「真実生成装置」ではなく「認知の揺らぎ生成器」である

4.2 Decision Layer（意味付け層）

現在は簡略実装されているが、本来の役割：

confidence → 数値正規化
contradiction → エントロピー化
severity → 影響係数化

役割定義：

LLM出力を物理エンジンが扱える形式へ変換する中間層

4.3 Cognitive State（認知状態）

構造：

LLMの解釈結果
ノードごとの推論状態
矛盾の保持

特徴：

「世界の状態」ではなく「世界の認知状態」

4.4 Threat Engine（認知→物理変換層）

最重要モジュール。

LLMの出力を以下に変換：

視界（FOV）
移動速度
空間ノイズ
プレイヤー視認距離

設計思想：

不確実性は“情報”ではなく“物理現象”として扱われる

4.5 Simulation Engine（ゲーム世界）

構成要素：

プレイヤー
セキュリティコア（敵AI）
ノード（BIO / MEC / CYB）
環境ノイズ

特徴：

状態はリアルタイムで変化
LLM推論結果が物理環境に反映される
4.6 Rendering Layer（可視化）

技術：

HTML5 Canvas

役割：

認知状態の視覚化
Threatの空間歪み表現
視界制限（confidence依存）

6. Key Innovation（本質的な新規性）

本システムの核心は以下にある：

6.1 LLM is not a tool, but a physical variable generator

従来：

LLM = 文章生成

本システム：

LLM = 世界状態変数生成器
6.2 Hallucination as System Feature

LLMの誤り（幻覚）はバグではなく：

ゲームメカニクスとして利用される

6.3 Inference Collapse Model

推論の揺らぎが以下を生成：

空間歪み
敵AI強化
視界制限

つまり：

認知の不安定性が物理現象に変換される

7. Architecture Nature（本システムの性質）

本システムは通常のソフトウェアではなく：

“動作するアーキテクチャ論文”

である。

特徴：

設計 → 実装ではなく
実装 → 設計抽出

という逆順構造を持つ。

8. Future Extensions（拡張予定）

9. Conclusion（結論）

本プロジェクトはゲームではなく、

LLMの認知構造が物理世界を生成する過程を観測するための実験系

である。

価値は完成度ではなく：

構造の成立
推論→物理変換の可視化
後付け設計によるアーキテクチャ生成

にある。
