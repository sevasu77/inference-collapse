# Future Improvements

## 概要

Inference Collapse は、**「LLM の認知状態をゲーム世界の物理法則へ変換する」**というコンセプトを検証するためのプロトタイプとして開発されました。

現在の実装でコアアイデアは実証できていますが、本プロジェクトはここで完成ではありません。

今後は、より保守性・拡張性・再利用性を高めながら、実験プラットフォームとして発展させていくことを目標としています。

---

# 今後の改善予定

## 1. FastAPI への移行

現在は Streamlit が UI とバックエンドの両方を担当しています。

将来的にはバックエンドを FastAPI として独立させ、

- `/inference`
- `/state`
- `/simulation`

などの API を提供する構成へ移行する予定です。

これにより、

- UI の自由な変更
- REST API 化
- 外部アプリケーションとの連携
- テストの容易化

が可能になります。

---

## 2. リアルタイム通信（WebSocket）

現在は画面更新ごとに状態を取得しています。

将来的には WebSocket を利用し、

- AI 推論結果
- ゲーム状態
- イベント
- Telemetry

をリアルタイムで同期できる構造を目指します。

---

## 3. マルチ LLM 対応

現在は Gemma を利用していますが、設計上は特定のモデルへ依存しない構造を目指しています。

将来的には、

- Gemma
- GPT 系モデル
- Claude
- その他のローカル LLM

を差し替え可能な設計へ発展させる予定です。

さらに、

- Investigator
- Adversary
- Observer

など複数の AI が同時に認知状態を持つマルチエージェント構成も検討しています。

---

## 4. Telemetry の強化

現在もゲーム中の情報を記録していますが、

今後は Telemetry をさらに発展させ、

- 推論履歴
- 状態変化
- Threat の変換結果
- シミュレーション結果

を可視化できるようにします。

これにより、

- デバッグ
- リプレイ
- Explainability（説明可能性）
- AI の振る舞い分析

へ活用できます。

---

## 5. リプレイ機能

ゲーム中の

- AI の認知
- State
- Threat
- Physics

を保存し、

後から同じ状況を再現できるリプレイ機能を追加する予定です。

これはデバッグだけでなく、研究用途にも利用できます。

---

## 6. より高度なシミュレーション

現在は Threat Engine が物理パラメータを生成しています。

今後は、

- 空間そのものの変形
- 認知によるルール変化
- エージェント同士の相互作用
- より複雑な環境シミュレーション

など、AI の認知状態が世界へ与える影響をさらに発展させる予定です。

---

## 7. 実験プラットフォームとしての発展

本プロジェクトの目的はゲームを完成させることではありません。

目指しているのは、

> **AI の認知状態がどのように世界へ影響するかを、安全かつ再現可能な形で観測・実験できるプラットフォーム**

を構築することです。

そのため、

- AI
- Decision Layer
- Game State
- Threat Engine
- Simulation
- UI

をそれぞれ独立したコンポーネントとして進化させ、新しい AI モデルやゲームシステムへ応用できるアーキテクチャを目指します。

---

# 将来像

現在の Inference Collapse は、コンテスト向けに開発されたプロトタイプです。

しかし、その過程で得られた知見を基に、今後はより拡張性の高い設計へ段階的に発展させていきます。

最終的な目標は、

> **AI の認知状態を、安全かつ制御可能な形でゲーム世界の物理法則へ変換する実験プラットフォーム**

を実現することです。
