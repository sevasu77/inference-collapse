# 05. Future Improvements

## 概要

Inference Collapse は、**「LLM の認知状態をゲーム世界の物理法則へ変換する」**というコンセプトを検証するためのプロトタイプとして開発されました。

現在の実装によってコアアイデアの実証は完了していますが、本プロジェクトはここで完成ではありません。

今後は、より保守性・拡張性・再利用性を備えた実験プラットフォームへ発展させることを目標としています。

---

# 開発方針

今後の改善は、単に機能を追加することが目的ではありません。

以下の設計方針を維持しながら、段階的にアーキテクチャを進化させていきます。

* AI とゲームロジックを疎結合に保つ
* State を唯一の状態管理層とする
* 各コンポーネントを独立して交換・拡張できる構造を維持する
* AI の認知状態を物理モデルへ変換するという基本思想を維持する

---

# 今後の改善予定

## 1. FastAPI によるバックエンド分離

現在は Streamlit が UI とバックエンドの両方を担当しています。

将来的にはバックエンドを FastAPI として独立させ、API を中心とした構成へ移行する予定です。

```
POST /inference

GET  /state

POST /simulation
```

これにより、

* UI の自由な変更
* REST API 化
* 外部アプリケーションとの連携
* テスト容易性の向上
* バックエンドの独立運用

が可能になります。

---

## 2. WebSocket によるリアルタイム同期

現在は画面更新のタイミングで状態を取得しています。

今後は WebSocket を利用し、

* AI 推論結果
* ゲーム状態
* Telemetry
* イベント情報

をリアルタイムで同期する構成を目指します。

これにより、

* より滑らかなゲーム体験
* 非同期推論との統合
* マルチクライアント対応

などが実現できます。

---

## 3. Multi-Agent Architecture

現在は単一の LLM が世界を認知しています。

将来的には複数の AI が同時に認知を行うマルチエージェント構成を検討しています。

例：

* Investigator Agent
* Adversary Agent
* Observer Agent

各エージェントは独立した認知状態を持ち、それぞれが同じゲーム世界に対して異なる解釈を行います。

その認知状態は Threat Engine を通して物理法則へ変換され、複数の AI が世界へ同時に影響を与えるシミュレーションを実現することを目指しています。

---

## 4. Telemetry の強化

現在もゲーム中の情報は記録していますが、今後は Telemetry をより発展させます。

記録対象には、

* 推論履歴
* State の変化
* Threat Engine の変換結果
* Simulation の実行結果

などを含め、可視化・分析できるようにする予定です。

これにより、

* デバッグ
* リプレイ
* Explainability（説明可能性）
* AI の振る舞い分析

へ活用できます。

---

## 5. リプレイ機能

AI の認知状態から物理世界が変化する一連の流れを保存し、後から同じ状況を再現できるリプレイ機能を追加する予定です。

保存対象には、

* Cognitive State
* Game State
* Threat Parameters
* Physics Parameters

などを含める予定です。

この機能はデバッグだけでなく、研究用途や AI の比較実験にも利用できます。

---

## 6. より高度なシミュレーション

現在は Threat Engine が比較的単純な物理パラメータを生成しています。

今後は、

* 空間そのものの変形
* 認知によるルール変更
* エージェント同士の相互作用
* より高度な環境シミュレーション

など、AI の認知状態が世界へ与える影響をさらに発展させる予定です。

---

# 長期ビジョン

Inference Collapse の目的は、ゲームを完成させることではありません。

目指しているのは、

> **「AI の認知状態が、どのように世界の振る舞いへ変換されるのか」を安全かつ再現可能な形で観測・実験できるプラットフォーム**

を構築することです。

そのため今後も、

* AI
* Decision Layer
* Game State
* Threat Engine
* Simulation Engine
* UI

を独立したコンポーネントとして進化させ、新しい AI モデルやゲームシステムへ応用できるアーキテクチャを目指します。

---

# 将来像

現在の Inference Collapse は、コンテスト向けに開発されたプロトタイプです。

しかし、その過程で得られた知見をもとに、今後はより拡張性・保守性・再利用性の高い設計へ段階的に発展させていきます。

最終的な目標は、

> **「AI の認知状態を、安全かつ制御可能な形でゲーム世界の物理法則へ変換する実験プラットフォーム」**

を実現することです。

このリポジトリは、その進化の過程そのものも記録し続けます。
