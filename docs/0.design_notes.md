🧠 AI Cognitive World Engine（仮）
概要

本プロジェクトは、

「AIの認知状態（確信度・幻覚・危険度）が、そのままゲーム世界の物理法則を変える」

というコンセプトを持つ、実験的ゲームエンジンです。

従来のゲームのように「難易度が固定されている」のではなく、
AIの推論結果そのものが世界のルールを変化させる構造になっています。

🏛️ システム設計思想

このシステムは単なるゲームではなく、

「AI・状態・物理・UIを明確に分離し、
それぞれの境界を制御可能にした実験的アーキテクチャ」

として設計されています。

目的は以下の通りです：

AIが世界を直接改変する構造の検証
状態管理の破綻（神オブジェクト化）の防止
リアルタイムゲームと非決定的AI推論の分離
再現可能なシミュレーション構造の構築

🧠 1. LLM層（認知・推論エンジン）

AIの「頭脳」に相当する層。

役割
証拠から状況を推理
世界の危険度を評価
矛盾や幻覚の検出
入出力

Input

{
  "sector": "BIO | MEC | CYB",
  "evidence": "観測データ"
}

Output

{
  "report": "推論結果",
  "confidence": 0.85,
  "severity": 0.7,
  "contradiction": false
}
💾 2. State Manager（世界の記録層）

ゲーム世界のセーブデータに相当する層。

役割
AI推論結果の保存
世界状態の管理
ログ・履歴の蓄積
状態構造
WorldState     → 物理的状態
AIState        → 推論結果
EngineState    → 物理演算状態
MetaState      → ログ・履歴
注意点

状態を単一オブジェクトにすると、

神オブジェクト化
デバッグ不能化
依存関係崩壊

が発生するため、必ず分離する。

🎮 3. Game Engine（物理世界）

役割

環境変化

❗ 現在の課題
1. 密結合問題

現在の構造は以下のようになっている：

LLM ─┐
     ├── STATE
UI ──┤
     ├── ENGINE
JS ──┘

→ 全層が直接干渉し、スパゲティ化リスクが高い

2. LLM依存リスク
出力の揺らぎで世界が崩壊
再現性の低下
デバッグ困難
🧩 設計上の解決方針
Stateの分割
State
 ├── WorldState
 ├── AIState
 ├── EngineState
 └── MetaState
API分離思想

すべての通信をAPI経由に統一：

/inference
/state/update
/simulate
🧾 ADR（設計意思決定）
なぜLLMとEngineを分けるのか

→ AIは非決定的、Engineは決定的
→ 混ぜるとリアルタイム性が崩壊するため

なぜStateを中央管理するのか

→ 直接干渉するとデバッグ不能になるため
→ 一度必ず「セーブデータ」に落とす構造にする
