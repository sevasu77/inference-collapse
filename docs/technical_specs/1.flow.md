# Flow

## 概要

このドキュメントでは、Inference Collapse におけるデータの流れと、各コンポーネントがどのように連携してゲーム世界を更新するのかを説明します。

01_architecture.md がシステム全体の構造を説明する設計図であるのに対し、本ドキュメントでは **実際の実行シーケンス** に焦点を当てます。

---

# 全体フロー

```text
User
  │
  ▼
UI Layer
  │
  ▼
AI Layer (Gemma)
  │
  ▼
Decision Layer
  │
  ▼
Game State
  │
  ▼
Threat Engine
  │
  ▼
Physics Parameters
  │
  ▼
Simulation Engine
  │
  ▼
Rendering (Canvas)
  │
  ▼
Player
```

Inference Collapse は、すべてのデータが一方向に流れる **Unidirectional Data Flow** を採用しています。

---

# 実行シーケンス

## ① User Input

プレイヤーがゲームへ入力を行います。

入力例

* Evidence（証拠）
* 調査対象
* ボタン操作
* ゲーム開始

UI は入力のみを担当し、ゲームロジックは保持しません。

---

## ② AI Inference

入力された情報を Gemma が解析します。

入力

```json
{
  "sector": "BIO",
  "evidence": "..."
}
```

出力

```json
{
  "report": "...",
  "confidence": 0.91,
  "severity": 0.72,
  "contradiction": false
}
```

この段階では、まだゲーム世界は変更されません。

LLM は **認知状態（Cognitive State）** を生成するだけです。

---

## ③ Decision Layer

LLM の出力をそのまま利用することはありません。

Decision Layer が以下を担当します。

* 型チェック
* バリデーション
* 数値の丸め
* 範囲外データの補正
* NormalizedInference への変換

```text
Raw JSON
      │
      ▼
Validation
      │
      ▼
NormalizedInference
```

この層が、AI の非決定的な出力からゲームを保護する「防波堤」となります。

---

## ④ Game State

正規化された推論結果を Game State に保存します。

```text
GameState

├── Environment
├── Cognition
├── Runtime
└── Telemetry
```

ここでは状態を保持するだけであり、ロジックは実行しません。

---

## ⑤ Threat Engine

Threat Engine は、本プロジェクトの中核となるコンポーネントです。

AI の認知状態を、ゲーム世界で利用できる物理パラメータへ変換します。

| Cognitive State | Physics Parameter  |
| --------------- | ------------------ |
| Confidence      | Enemy Speed        |
| Severity        | Spatial Distortion |
| Hallucination   | Field of View      |
| Contradiction   | Entropy            |

この層によって、

```text
AIの認知
      ↓
物理法則
```

という変換が実現されます。

---

## ⑥ Simulation Engine

Simulation Engine は Physics Parameters のみを参照します。

更新対象

* Enemy
* Player
* Timer
* Collision
* Vision
* Effects

Simulation Engine は、

* LLM
* Decision Layer

の存在を知りません。

そのため、AI モデルを変更してもゲームロジックを修正する必要はありません。

---

## ⑦ Rendering

更新された世界を描画します。

描画対象

* HTML5 Canvas
* HUD
* Status Panel
* Glitch Effect
* Field of View

Rendering は表示のみを担当し、状態は変更しません。

# 設計上の制約

以下の構造は禁止します。

❌ LLM → Simulation Engine

❌ UI → Game State の直接変更

❌ Rendering → Game Logic

❌ JavaScript → AI State

これらは責務の混在や密結合を招きます。

---

# 設計原則

本プロジェクトでは、各コンポーネントが単一の責務のみを持ちます。

| Component         | Responsibility |
| ----------------- | -------------- |
| UI                | 入力・表示          |
| AI Layer          | 推論             |
| Decision Layer    | 正規化・検証         |
| Game State        | 状態保持           |
| Threat Engine     | 認知 → 物理変換      |
| Simulation Engine | 世界更新           |
| Rendering         | 描画             |

この責務分離により、各コンポーネントは独立して開発・テスト・拡張できます。

---

# コアコンセプト

Inference Collapse のデータフローは、

```text
LLM
    ↓
Cognitive State
    ↓
Physics
    ↓
Simulation
    ↓
Rendering
    ↓
Player Experience
```

という一方向のパイプラインによって構成されています。

AI がゲーム世界を直接制御するのではなく、**AI の認知状態を物理法則へ変換すること**が、本プロジェクトの最も重要な設計思想です。
